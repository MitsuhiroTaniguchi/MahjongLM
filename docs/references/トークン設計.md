# トークン設計

## 出力フォーマット

`tokenize_tenhou.py` は1ゲームを1レコードで出力する。

```json
{
  "source_zip": ".../data2023.zip",
  "game_id": "...json",
  "tokens": ["game_start", "...", "game_end"]
}
```

## トークン一覧

### 対局ライフサイクル

- `game_start`: 対局開始
- `game_end`: 対局終了

### 局開始情報

- `round_start`: 局開始
- `bakaze_{zhuangfeng}`: 場風
- `kyoku_{jushu}`: 局番号（東1, 東2 ... に対応）
- `honba`: 本場ヘッダ。直後に本場料合計（`changbang * 100` 点）を表す点棒トークン列が続く
- `riichi_sticks`: 供託リーチ棒ヘッダ。直後に供託合計（`lizhibang * 1000` 点）を表す点棒トークン列が続く
- `dora`: ドラ表示牌ヘッダ。直後に牌種トークンが続く
- `score_{seat}`: 局開始時点の持ち点ヘッダ
- `TENBO_PLUS` / `TENBO_MINUS` / `TENBO_ZERO`: 点棒値の符号
- `TENBO_100`, `TENBO_200`, `TENBO_300`, `TENBO_500`, `TENBO_1000`, `TENBO_2000`, `TENBO_3000`, `TENBO_5000`, `TENBO_10000`, `TENBO_20000`, `TENBO_30000`: 点棒単位（値を分解して表現）
- `haipai_{seat}`: 配牌ヘッダ。直後に牌種トークンが続く
- `m0`-`m9`, `p0`-`p9`, `s0`-`s9`, `z1`-`z7`: 汎用牌種トークン

### 自摸・打牌

- `draw_{seat}_{tile}`: 自摸
- `discard_{seat}_{tile}`: 打牌

後続トークン（手出し/ツモ切りが両方とも候補手の場合のみ）:
- `tedashi`: 手出しフラグ
- `tsumogiri`: ツモ切りフラグ

### 自己意思決定

- 候補: `opt_self_{seat}_{action}`（その手番で選択可能な自己行動）
- 採用: `take_self_{seat}_{action}`（実際に選んだ自己行動）
- 不採用: `pass_self_{seat}_{action}`（選ばなかった自己行動）

`haipai_{seat}` は seat ごとに 1 回だけ出て、その後にその配牌を表す牌種トークン列が続く。
`ankan` / `kakan` では、各 token の直後に対象牌種を表す汎用牌種トークンが続く。
`opt_self_{seat}_{ankan|kakan}` は 1 回だけ出て、その後に候補牌種列が続く。

`action`:
- `tsumo`: ツモ和了
- `riichi`: 立直
- `ankan`: 暗槓
- `kakan`: 加槓
- `kyushukyuhai`: 九種九牌流局宣言

### 反応意思決定

- 候補: `opt_react_{seat}_{action}`（他家打牌/加槓への反応候補）
- 採用: `take_react_{seat}_{action}`（実際に選んだ反応）
- 不採用: `pass_react_{seat}_{action}_{reason}`（選ばれなかった反応）
- チー成立位置: `chi_pos_{low|mid|high}`（鳴いた牌が順子の下/中/上か）
- 赤牌選択: `red_{chi|pon}_{used|not_used}`（赤を使う選択肢が実在した場合のみ）

`action`:
- `ron`: ロン和了
- `chi`: チー
- `pon`: ポン
- `minkan`: 大明槓

`reason`:
- `voluntary`: 本人が見送り
- `forced_priority`: 優先順位で失効

### 流局・点棒変動

- `pingju_{reason}`: 流局種別
- `score_delta_{seat}`: その局での点棒増減ヘッダ
- `TENBO_*`: `score_delta_{seat}` の直後に続く増減値本体

### 不明イベント

- `event_unknown_{key}`: 未対応イベントキー
