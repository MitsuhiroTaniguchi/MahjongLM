# トークン設計

## 出力単位

### 単一 view

`tokenize_tenhou.py` は 1 game を 1 record として出力する。

```json
{
  "source_zip": ".../data2023_sample_v1.zip",
  "game_id": "...json",
  "tokens": ["game_start", "...", "game_end"]
}
```

### multiview

同じ game から次を生成する。

- complete (完全情報) view 1 本
- imperfect (不完全情報) view を各 seat ごとに 1 本

artifact 名は `game_id__complete.*` または `game_id__player_{seat}.*`。

注意:

- artifact 上の `player_{seat}` は round seat
- token 内の `view_imperfect_{player}` の `player` は `qijia` 基準

## トークン一覧

### 対局説明

- `game_start`
- `view_complete` / `view_imperfect_{player}`
- `rule_player_3` / `rule_player_4`
- `rule_length_tonpu` / `rule_length_hanchan`
- `game_end`

### 局開始情報

- `round_start`
- `bakaze_{zhuangfeng}`
- `kyoku_{jushu}`
- `honba`
- `riichi_sticks`
- `dora`
- `ura_dora`
- `score_{seat}`
- `final_score_{seat}`
- `TENBO_PLUS` / `TENBO_MINUS` / `TENBO_ZERO`
- `TENBO_100`, `TENBO_200`, `TENBO_300`, `TENBO_500`, `TENBO_1000`, `TENBO_2000`, `TENBO_3000`, `TENBO_5000`, `TENBO_10000`, `TENBO_20000`, `TENBO_30000`
- `haipai_{seat}` / `hidden_haipai_{seat}`
- `m0`-`m9`, `p0`-`p9`, `s0`-`s9`, `z1`-`z7`

### 自摸・打牌

- `draw_{seat}_{tile}`
- `draw_{seat}_hidden`
- `discard_{seat}_{tile}_tedashi`
- `discard_{seat}_{tile}_tsumogiri`

打牌トークン自体に手出し/ツモ切りを融合する。各打牌は必ずどちらか一方で表現し、独立した `tedashi` / `tsumogiri` トークンは使わない。

### 自己意思決定

- `opt_self_{seat}_{action}`
- `take_self_{seat}_{action}`
- `pass_self_{seat}_{action}`

`action`:

- `tsumo`
- `riichi`
- `ankan`
- `kakan`
- `kyushukyuhai`
- `penuki`

`take_self_{seat}_{ankan|kakan}` は直後に対象牌トークンが続く。`opt_self_{seat}_{ankan|kakan}` は 1 回だけ出て、後ろに候補牌列が続く。

### 反応意思決定

- `opt_react_{seat}_{action}`
- `take_react_{seat}_{action}`
- `pass_react_{seat}_{action}_{reason}`
- `chi_pos_{low|mid|high}`
- `red_{chi|pon}_{used|not_used}`

`action`:

- `ron`
- `chi`
- `pon`
- `minkan`

`reason`:

- `voluntary`
- `forced_priority`

### 和了・流局・点棒変動

- `pingju_{reason}`
- `yaku_{name}`
- `han_{N}`
- `fu_{N}`
- `yakuman_{N}`
- `score_delta_{seat}`
- `rank_{seat}_{place}`
- `final_rank_{seat}_{place}`

`fu_{N}` は `25` と `20..140` の 10 刻みを使う。

和了時は

- `take_self_*_tsumo` または `take_react_*_ron`
- `yaku_*`
- `han_*` / `fu_*` / `yakuman_*`
- `score_delta_*`

の順。

`han_*` は `13` を上限に正規化する。`yakuman_*` は `damanguan` が立っている役満手だけに使う。

`yaku_dora` / `yaku_ura_dora` / `yaku_aka_dora` は成立翻数ぶん繰り返す。

### 手牌公開

和了時:

- `opened_hand_{winner}` を役詳細の先頭に出す
- 直後に concealed hand の牌トークンだけを並べる
- 副露牌や和了牌は含めない

流局時:

- 流局成立証明として手牌の公開が意味を持つケースに限り、`pingju_{reason}` の直後に `opened_hand_{seat}` を並べる

成立証明対象:

- `pingju_ryukyoku`
- `pingju_kyushukyuhai`
- `pingju_suuchariichi`
- `pingju_sanchahou`

対象でないもの:

- `pingju_nagashimangan`
- `pingju_sufurenda`
- `pingju_sukantsu`

## 順位

局終了後:

- `rank_0_* .. rank_{seat_count-1}_*`

game 終了直前:

- `final_score_0 .. final_score_{seat_count-1}`
- `final_rank_0_* .. final_rank_{seat_count-1}_*`

同点解決:

- 局中順位は seat rotation を考慮
- 最終順位は top-level `qijia` 基準

## multiview の隠し方

imperfect view では viewer 以外について:

- 初期手牌: `hidden_haipai_{seat}` 1 個
- 自摸: `draw_{seat}_hidden`
- 他家 option / pass: 非表示

これにより viewer 目線で見える情報と見えない情報を時系列で正確に表現できる。
